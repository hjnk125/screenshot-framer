# SEO 강화 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** index.html에 SEO 메타태그 전체 추가 + sitemap.xml / robots.txt 생성으로 검색 노출과 SNS 링크 미리보기를 완성한다.

**Architecture:** 정적 HTML 파일과 public 디렉터리 파일만 수정. 빌드 파이프라인 변경 없음. og-image.png는 이미 `public/`에 추가됨.

**Tech Stack:** HTML, XML, Vite (public/ 파일은 빌드 시 그대로 복사됨)

---

## 파일 구조

| 파일 | 작업 |
|------|------|
| `index.html` | title, description, canonical, OG, Twitter Card, JSON-LD 추가 |
| `public/sitemap.xml` | 신규 생성 |
| `public/robots.txt` | 신규 생성 |
| `cspell.json` | `changefreq`, `sitemap`, `pricecurrency` 등 단어 추가 |

### Task 1: sitemap.xml / robots.txt 생성

**Files:**
- Create: `public/sitemap.xml`
- Create: `public/robots.txt`

- [ ] **Step 1: sitemap.xml 생성**

`public/sitemap.xml` 내용:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://screenshot-framer.vercel.app/</loc>
    <changefreq>monthly</changefreq>
    <priority>1.0</priority>
  </url>
</urlset>
```

- [ ] **Step 2: robots.txt 생성**

`public/robots.txt` 내용:

```
User-agent: *
Allow: /
Sitemap: https://screenshot-framer.vercel.app/sitemap.xml
```

- [ ] **Step 3: cspell 단어 추가**

`cspell.json`의 `words` 배열에 추가:

```json
"changefreq",
"sitemap",
"urlset"
```

- [ ] **Step 4: 브라우저에서 확인**

`npm run dev` 후:
- `http://localhost:5173/sitemap.xml` → XML 내용 출력 확인
- `http://localhost:5173/robots.txt` → 텍스트 출력 확인

- [ ] **Step 5: 커밋**

```bash
git add public/sitemap.xml public/robots.txt cspell.json
git commit -m "feat: sitemap.xml, robots.txt 추가"
```

### Task 2: index.html 메타태그 추가

**Files:**
- Modify: `index.html`

- [ ] **Step 1: `<head>` 안에 메타태그 전체 추가**

`index.html`을 아래와 같이 교체:

```html
<!doctype html>
<html lang="ko">
  <head>
    <meta charset="UTF-8" />
    <link rel="icon" type="image/svg+xml" href="/favicon.svg" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />

    <!-- Primary -->
    <title>Screenshot Framer — Device & Browser Frame Tool</title>
    <meta name="description" content="스크린샷을 iPhone, MacBook, Chrome, Safari 프레임으로 감싸 고품질 PNG로 export. Wrap screenshots in device or browser frames and export as high-quality PNG." />
    <link rel="canonical" href="https://screenshot-framer.vercel.app/" />

    <!-- Open Graph -->
    <meta property="og:type" content="website" />
    <meta property="og:url" content="https://screenshot-framer.vercel.app/" />
    <meta property="og:title" content="Screenshot Framer — Device & Browser Frame Tool" />
    <meta property="og:description" content="스크린샷을 iPhone, MacBook, Chrome, Safari 프레임으로 감싸 고품질 PNG로 export. Wrap screenshots in device or browser frames and export as high-quality PNG." />
    <meta property="og:image" content="https://screenshot-framer.vercel.app/og-image.png" />
    <meta property="og:locale" content="ko_KR" />
    <meta property="og:locale:alternate" content="en_US" />

    <!-- Twitter Card -->
    <meta name="twitter:card" content="summary_large_image" />
    <meta name="twitter:title" content="Screenshot Framer — Device & Browser Frame Tool" />
    <meta name="twitter:description" content="스크린샷을 iPhone, MacBook, Chrome, Safari 프레임으로 감싸 고품질 PNG로 export." />
    <meta name="twitter:image" content="https://screenshot-framer.vercel.app/og-image.png" />

    <!-- JSON-LD -->
    <script type="application/ld+json">
    {
      "@context": "https://schema.org",
      "@type": "WebApplication",
      "name": "Screenshot Framer",
      "url": "https://screenshot-framer.vercel.app/",
      "description": "Wrap screenshots in device or browser frames and export as high-quality PNG.",
      "applicationCategory": "DesignApplication",
      "operatingSystem": "Web",
      "offers": {
        "@type": "Offer",
        "price": "0",
        "priceCurrency": "USD"
      }
    }
    </script>
  </head>
  <body>
    <div id="root"></div>
    <script type="module" src="/src/main.tsx"></script>
  </body>
</html>
```

- [ ] **Step 2: 브라우저에서 확인**

`npm run dev` 후:
- 브라우저 탭 타이틀이 "Screenshot Framer — Device & Browser Frame Tool"인지 확인
- DevTools → Elements → `<head>` 안에 og:image, twitter:card 등 태그 확인

- [ ] **Step 3: OG 미리보기 확인 (선택)**

[https://opengraph.xyz](https://opengraph.xyz) 또는 [https://cards-dev.twitter.com/validator](https://cards-dev.twitter.com/validator) 에서 배포 URL 검증.
(배포 후에만 가능)

- [ ] **Step 4: cspell 단어 추가**

`cspell.json`의 `words` 배열에 추가:

```json
"pricecurrency",
"priceCurrency"
```

- [ ] **Step 5: 커밋**

```bash
git add index.html cspell.json
git commit -m "feat: SEO 메타태그 추가 — OG, Twitter Card, JSON-LD, canonical"
```
